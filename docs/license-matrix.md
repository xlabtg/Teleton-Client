# Upstream License Matrix

This matrix records the foundation-phase license review for planned upstream dependencies and reference implementations. It is an engineering inventory for release planning, not a substitute for human legal review.

Verified on 2026-04-26 UTC from GitHub license metadata and upstream license or README files. Before a release, every selected upstream must be rechecked at the exact tag, commit, or package version being shipped.

## Review Policy

- Reference-only entries are allowed for product, architecture, and behavior research, with no code copy, asset copy, static linking, dynamic linking, generated source import, or vendored snippets.
- Copyleft entries remain outside distributed Teleton Client artifacts unless a human legal reviewer approves the exact reuse plan and source publication obligations.
- Permissive entries still require preserving upstream copyright notices, license text, package version, source URL, and local patch notes in third-party notices.
- Every bundled binary, native library, WebAssembly module, generated SDK, or app-store package must record the upstream revision and any local modifications before release readiness sign-off.
- Human legal review is required before release readiness for TDLib packaging, Telegram reference reuse, Teleton Agent bundling, TON SDK selection, and any dependency with unclear or mixed licensing.

## Release Sign-Off Checklist

Before any public package, app build, or release candidate, the reviewer must confirm:

1. The selected dependency list matches this matrix or this matrix has been updated for new upstreams.
2. Every shipped dependency has an exact version, source URL, license identifier, and license text in the release records.
3. Every local patch is documented with a source publication decision.
4. GPL and LGPL entries have either remained reference only with no code copy or have a written copyleft compliance plan.
5. Third-party notices preserve MIT, BSL-1.0, Apache-2.0, GPL, LGPL, and upstream exception text where required.
6. Human legal review has signed off on TDLib packaging, Telegram references, Teleton Agent bundling, TON SDKs, and app-store distribution constraints.
